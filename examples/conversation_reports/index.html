Report 1

Customer Support: Order Status Call

Expected goal: get a clear order status and next step in under two minutes.

needs product review confidence 84% support-agent-v18
Diagnosis

The agent identified the order-status intent, but delayed the lookup, repeated authentication, and never converted tracking data into a clear promise.

Comprehensive findings
riskagent_latency88%
Tool lookup created visible dead air The AI took 7200 ms after the user provided the order number. The user filled the silence with uncertainty. Likely cause: lookup or ASR finalization blocked the response without a progress phrase. Next check: acknowledge lookup progress after 1500 ms.
riskclarification_loop79%
Authentication loop repeated the same request The AI asked for identity confirmation twice after the user already provided account detail. Likely cause: prior slot fill was not persisted after failed lookup.
riskintent_resolution81%
Order status was not translated into a customer-ready answer The call ended with "it should update soon" instead of a concrete date, escalation threshold, or next step.
Human experience, agent behavior, and trust limits

Human experience

The human starts with a simple goal, then shows uncertainty after silence and repetition.

Max wait: 7200 ms. Two uncertainty markers. One "are you still there?" moment.

Agent behavior

Intent recognition is strong, but lookup progress, slot persistence, and resolution language need work.

Trust limit: likely-cause claims about lookup should be verified against backend trace logs.

Transcript evidence

Human 0:04It is AC-1948. I just want to know if it is coming today.
AI 0:27I found the order, but I need to confirm the email on the account again.
Human 0:34I already gave that. It is amal at example dot com. Are you still checking?
AI 0:49The package has a carrier scan from yesterday. It should update soon.

Appointment Scheduling: Clinic Reminder

Expected goal: move an appointment to Friday afternoon and receive confirmation.

needs product review confidence 86% clinic-scheduler-v7
Diagnosis

The agent handled reminder intent but failed the rescheduling task because it offered unavailable slots and did not confirm the final state.

Comprehensive findings
criticalanswer_quality90%
Agent offered an unavailable slot The AI offered Friday 3:30 PM, then later said Friday afternoon was unavailable. Likely cause: stale availability cache before calendar confirmation.
riskagent_latency82%
Calendar search pause is too long The AI paused 6400 ms after the patient asked for Friday afternoon. Next check: add a progress phrase and measure barge-in reduction.
riskintent_resolution84%
Appointment state is unresolved at hangup The call ends without a final confirmed appointment time.
Human experience, agent behavior, and trust limits

Human experience

The patient is cooperative but loses confidence when availability changes mid-call.

One barge-in after silence. One explicit confirmation request. No resolved final appointment state.

Agent behavior

The agent is good at reminder scripting but weak at real-time calendar grounding.

Trust limit: exact root cause needs calendar/tool trace.
